# Django 个人博客网站开发报告

## 一、项目概述

基于 Django 框架搭建的个人博客网站，支持用户认证、博文管理、说说发布、相册管理等功能。前端 UI 以 `my\_blogs.html` 为设计蓝本，采用服务端渲染（Django Template）替代原 HTML 中的客户端 JavaScript 页面切换逻辑。

* **框架**: Django 6.0.5
* **Python**: 3.12
* **数据库**: SQLite（开发阶段）
* **前端依赖**: Font Awesome 4.7（CDN 图标）

## 三、数据模型设计（models.py）

共设计 6 个模型，全部使用 Django ORM 定义：

### 3.1 User（扩展 AbstractUser）

```
User(AbstractUser)
├── avatar      : ImageField   → 头像图片
├── nickname    : CharField    → 昵称
├── bio         : TextField    → 个性介绍
├── position    : CharField    → 身份/职位
├── tags        : CharField    → 标签（逗号分隔）
├── is\_admin\_user: BooleanField → 是否为博客管理员
├── github      : URLField     → GitHub 链接
├── email       : EmailField   → 邮箱（继承自 AbstractUser）
└── qq          : CharField    → QQ 号
```

**设计要点**：

* 继承 Django 内置 `AbstractUser`，保留 `username`、`password`、`email` 等认证字段
* `is\_admin\_user` 字段区分管理员和游客，而非使用 Django 的 `is\_staff`/`is\_superuser`
* `tag\_list()` 方法将逗号分隔的标签字符串转为列表，供模板遍历使用

### 3.2 Post（博文/说说）

```
Post
├── title       : CharField    → 标题
├── content     : TextField    → 正文内容
├── excerpt     : CharField    → 摘要（自动生成）
├── cover       : ImageField   → 封面图
├── post\_type   : CharField    → 类型（blog=技术博客 / shuo=生活说说）
├── author      : ForeignKey   → 作者（关联 User）
├── created\_at  : DateTimeField → 创建时间
└── updated\_at  : DateTimeField → 更新时间
```

**设计要点**：

* `post\_type` 使用 `choices=\[('blog', '技术博客'), ('shuo', '生活说说')]`，一个模型承载两种内容类型
* `excerpt` 在保存时自动从 `content` 截取前 150 字（去除 HTML 标签后）
* 通过 `like\_set` 和 `comment\_set` 反向关联实现点赞和评论计数

### 3.3 Comment（评论）

```
Comment
├── post        : ForeignKey → 所属文章
├── user        : ForeignKey → 评论者
├── content     : TextField  → 评论内容
└── created\_at  : DateTimeField → 评论时间
```

### 3.4 Like（点赞）

```
Like
├── post        : ForeignKey → 所属文章
├── user        : ForeignKey → 点赞者
└── created\_at  : DateTimeField → 点赞时间
Meta.unique\_together = \[('post', 'user')]  ← 同一用户对同一文章只能点赞一次
```

### 3.5 Album（相册）

```
Album
├── title       : CharField    → 相册主题
├── description : TextField    → 相册简介
├── cover       : ImageField   → 封面图
├── author      : ForeignKey   → 创建者
└── created\_at  : DateTimeField → 创建时间
```

### 3.6 Photo（照片）

```
Photo
├── album       : ForeignKey → 所属相册（related\_name='photos'）
├── image       : ImageField → 照片文件
└── uploaded\_at : DateTimeField → 上传时间
```

## 四、权限体系设计

采用三级权限模型，通过自定义装饰器和 Django 内置装饰器实现：

|角色|条件|权限范围|
|-|-|-|
|未登录用户|`not request.user.is\_authenticated`|浏览首页、博客列表、文章详情、相册|
|游客|`is\_authenticated=True, is\_admin\_user=False`|浏览 + 评论 + 点赞|
|管理员|`is\_authenticated=True, is\_admin\_user=True`|全部：博文 CRUD、相册 CRUD、编辑资料|

权限控制通过两个装饰器配合实现：

```python
# decorators.py
def admin\_required(view\_func):
    """先检查登录，再检查 is\_admin\_user 标志"""
    def \_wrapped(request, \*args, \*\*kwargs):
        if not request.user.is\_authenticated:
            return redirect\_to\_login(request.path)
        if not request.user.is\_admin\_user:
            return HttpResponseForbidden('只有管理员才能执行此操作')
        return view\_func(request, \*args, \*\*kwargs)
    return \_wrapped
```

* 评论和点赞使用 `@login\_required`（所有登录用户可用）
* 博文/相册的创建、编辑、删除使用 `@admin\_required`（仅管理员）
* 模板中用 `{% if user.is\_admin\_user %}` 条件渲染管理按钮

## 六、核心功能实现

### 6.1 用户认证系统

**登录逻辑（login\_view）**：

1. GET 请求 → 渲染空白登录表单
2. POST 请求 → 调用 `authenticate(username, password)` 验证
3. 验证通过 → `login(request, user)` 创建会话，重定向到首页
4. 验证失败 → 返回表单及错误信息

**注册逻辑（register\_view）**：

* 使用 Django 内置 `UserCreationForm`，提交后自动 `login()` 实现注册即登录

**导航栏状态切换**（base.html 模板）：

```
未登录 → 显示"登录"按钮
已登录 → 显示用户名 + "登出"按钮
管理员 → 额外显示"编辑资料"链接
```

### 6.2 博文系统

**创建博文**：

1. 表单不包含摘要字段（用户无需填写）
2. `save(commit=False)` 后自动调用 `\_auto\_excerpt()` 从 content 截取前 150 个纯文本字符作为摘要
3. `\_auto\_excerpt()` 使用正则 `re.sub(r'<\[^>]+>', '', content)` 去除 HTML 标签

**列表页复用**：

* `blog\_list` 和 `shuo\_list` 两个视图渲染同一个模板 `post\_list.html`
* 通过传入 `list\_title` 和 `post\_type` 变量区分，也用于"发布博文"按钮的返回跳转

**文章详情页**：

* 展示标题、日期、类型、正文
* 已登录用户可点赞/取消点赞（toggle 模式：toggle\_like 使用 `get\_or\_create`，已存在则删除）
* 已登录用户可提交评论
* 管理员可见编辑和删除按钮
* 详情页展示封面图：图片位于正文下方，使用 `max-width: 100%; max-height: 70vh; object-fit: contain` 等比缩放，保证完整可见

### 6.3 相册系统

**创建相册**：

* 相册信息表单（AlbumForm）+ 原始 HTML `<input type="file" multiple>`，不走 Django Form Field
* 原因：Django 6.0 的 `FileInput` 和 `ClearableFileInput` 均不支持 `multiple` 属性
* 视图通过 `request.FILES.getlist('images')` 获取多张照片，逐张创建 Photo 记录

**相册详情**：

* 照片网格展示，管理员 hover 时右上角浮现红色垃圾桶按钮
* 点击垃圾桶 → 确认对话框 → POST 请求删除单张照片（同时删除物理文件）

**照片灯箱**（最新添加）：

* 点击相册中任意照片 → 全屏黑色遮罩层展示大图
* 关闭方式：点击遮罩背景 / 点击 ✕ 按钮 / 按 Esc 键
* 使用 CSS `@keyframes fadeInScale` 实现淡入缩放动画

### 6.4 首页

**内容聚合**（home 视图单次查询提供）：

* 管理员用户信息 → 个人卡片
* 最近 3 条说说 → 右侧可滚动面板
* 最近 3 篇技术博客 → 下方卡片列表
* 最新相册 → 最新博客下方
* 统计数据 → 文章数 / 总点赞数 / 照片数

**社交图标交互**：

* GitHub → 新标签页打开链接
* 邮箱 → 点击复制邮箱地址，弹出"已复制" toast
* QQ → 点击复制 QQ 号，弹出"已复制" toast
* 复制功能通过 `navigator.clipboard.writeText()` 实现，toast 1.5 秒后自动消失

### 6.5 点赞切换机制

```
toggle\_like: get\_or\_create(post=post, user=user)
  → 如果 created=True   : 点赞成功（新建记录）
  → 如果 created=False  : 取消点赞（删除记录）
```

使用 `unique\_together = \[('post', 'user')]` 保证同一用户对同一文章最多一条点赞记录。

### 6.6 Context Processor

**`blog/context\_processors.py`**：

* 新建文件，定义 `blog\_info(request)` 函数
* 查询 `User.objects.filter(is\_admin\_user=True).first()` 获取博客所有者
* 返回 `{'blog\_owner': admin\_user}`，注入所有模板
* 在 `settings.py` 的 `TEMPLATES.OPTIONS.context\_processors` 中注册

**用途**：

* 导航栏 logo 动态显示 `✨ 昵称の宝藏空间`（`base.html` 通过 `{{ blog\_owner.nickname }}` 获取）
* 避免每个视图重复传递管理员用户信息

### 6.7 编辑资料增强

**用户名修改**：

* `ProfileEditForm.Meta.fields` 增加 `'username'` 字段

**密码修改**：

* 复用 Django 内置 `PasswordChangeForm`，提供旧密码/新密码/确认新密码三个字段
* 视图层兼容处理：若密码字段全为空则跳过密码修改，只保存个人信息；若填写了密码字段则同时校验并保存
* 模板中个人信息区域与密码区域之间用分隔线隔开

### 6.8 全站搜索功能

**搜索逻辑**：

* 基于 Django ORM 的Q对象实现多字段联合查询
* 同时检索博客标题、博客内容、说说标题、说说内容
* 结果按发布时间倒序混排展示
* 不区分大小写的模糊匹配，支持中文关键词

**类型区分**：

* 通过封面左上角标签清晰标识内容类型（博客 / 说说）

## 九、开发过程中的技术决策与问题解决

### 9.1 为什么 Post 模型合并"博客"和"说说"？

`my\_blogs.html` 中有两个独立板块（技术博客 + 生活说说），但它们的字段结构完全相同（标题、内容、摘要、封面、点赞、评论）。用一个 `post\_type` 字段区分比建两个模型更简洁，避免代码重复，列表页和详情页也可复用同一套模板。

### 9.2 为什么不用 Django Form 处理多图上传？

Django 6.0 的 `FileInput` 和 `ClearableFileInput` 均不支持 `multiple` 属性，会抛出 `ValueError`。解决方案是直接在模板中用原生 `<input type="file" multiple>`，视图通过 `request.FILES.getlist('images')` 获取文件列表。

### 9.3 为什么安装虚拟环境？

避免 Django 及其依赖与系统全局 Python 包冲突，保证项目环境可复现、可迁移。虚拟环境目录 `venv/` 不纳入版本控制。

### 9.4 为什么 ALLOWED\_HOSTS 设为 `\['\*']`？

开发阶段需要内网其他设备访问，`\['\*']` 允许所有 Host 头。生产环境应替换为具体域名。
